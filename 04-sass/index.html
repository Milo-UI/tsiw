3) Mixins

Utilissimo

Un mixin è una “ricetta” riutilizzabile. Qui lo usiamo per responsive e focus ring. 

@include up(md) { ... }

Prova tab ↹ su di me

4) Functions

Molto utile

Una function Sass restituisce un valore (es. conversioni come px → rem). È perfetta per calcoli puliti e riutilizzabili. 

// _functions.scss
@function rem($px, $base: 16) {
    @return ($px / $base) * 1rem;
}

// uso:
.title { font-size: rem(24); } // 1.5rem
rem(24) → 1.5rem rem(100) → 6.25rem

5) Maps

Design system

Le mappe sono coppie chiave → valore: utili per centralizzare configurazioni (spaziature, breakpoints, colori) e generare CSS con loop. 

// _variables.scss
$breakpoints: (sm: 576px, md: 768px, lg: 992px);
$spacers: (1: .25rem, 2: .5rem, 3: .75rem, 4: 1rem);

// leggere un valore:
@use "sass:map";
$md: map.get($breakpoints, md); // 768px

// loop per generare classi:
@each $k, $v in $spacers {
    .p-#{$k} { padding: $v; }
}
map.get() @each + utilities config centralizzata

4) @extend

Occhio

@extend riusa uno “scheletro” di stile. È comodo, ma può complicare il CSS finale se abusato. 

%pill { border-radius: 999px; }
.badge { @extend %pill; }
Sono una pill Anch’io

5) Loop (utilities)

Wow effect

Con un loop generiamo classi tipo .mt-2, .p-3 ecc. Una mini “Bootstrap” fatta in Sass. 

@each $k, $v in $spacers {
    .mt-#{$k} { margin-top: $v; }
}
